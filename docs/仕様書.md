# SampleELT 仕様書

## 概要

SampleELT は、Pentaho Spoon に類似したビジュアル ETL（Extract / Transform / Load）ツールです。
Oracle・MySQL データベースおよびファイル（Excel / CSV / TXT / TSV）を対象に、データの抽出・変換・出力をドラッグ＆ドロップで設計・実行できます。
複数のパイプラインをまとめて順次実行する **ジョブ機能** と、定時・定期の **スケジュール実行機能** を備えます。

## アーキテクチャ

### データフロー（パイプライン）

```csharp
Task<List<Dictionary<string, object?>>> ExecuteAsync(
    List<Dictionary<string, object?>> inputData,
    IProgress<string> progress,
    CancellationToken ct)
```

- **inputData** : 前ステップ（単一）の出力行リスト
- **AllInputStreams** : 複数入力時（MergeJoin 用）— `AllInputStreams[0]` = Left、`[1]` = Right
- Output ステップは inputData をそのまま返す（後続ステップへのパススルー）

### 実行エンジン

`ExecutionEngine` : Kahn's Algorithm でトポロジカルソート → 依存順に非同期実行。  
`JobExecutor` : ジョブ内の `JobStep` を `Order` 順に並べ `ExecutionEngine` を順次呼び出す。`ContinueOnError = false` のステップでエラーが発生した場合はジョブを中断する。

### DB 接続設定

`ConnectionRegistry`（シングルトン）が全接続設定を保持し `connections.json` に永続化。各ステップは `Settings["ConnectionId"]` (Guid 文字列) で接続を参照する。

## ステップ仕様

### Input グループ

#### DB Input（統合 DB 入力）

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID（ConnectionRegistry から選択）|
| SQL | 実行する SELECT 文 |
| ExecuteEachRow | `true`：前ステップの各行を `?` パラメータとして渡し行ごとに実行 |

#### Excel Input

| 設定項目 | 説明 |
|---|---|
| FilePath | Excel ファイルパス (.xlsx) |
| SheetName | 読み込むシート名 |
| HasHeader | 1行目をヘッダーとして扱う（デフォルト: true）|

---

### Output グループ

#### DB Output（統合 DB 出力）

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| TableName | 出力先テーブル名 |
| Mode | INSERT / UPSERT |
| CommitSize | N 件ごとにコミット（デフォルト: 100、0 = 最後に一括コミット）|

#### File Output（ExcelOutput ステップ）

| 設定項目 | 説明 |
|---|---|
| FilePath | 出力ファイルパス（**日時トークン使用可**）|
| Format | `Excel` / `CSV` / `TSV` / `TXT` |
| SheetName | シート名（Excel のみ）|
| Delimiter | 区切り文字（CSV/TXT: `,`  TSV: `\t` ）|
| Encoding | `UTF-8` / `UTF-8 BOM` / `Shift-JIS` |
| IncludeHeader | ヘッダー行を出力する（デフォルト: true）|

**日時トークン**

ファイルパス内の `{フォーマット}` を実行時の `DateTime.Now` で展開する。C# 標準日時フォーマット文字列をすべて使用可能。

| トークン例 | 展開例 |
|---|---|
| `{yyyyMMdd}` | `20260403` |
| `{yyyy-MM-dd}` | `2026-04-03` |
| `{HHmmss}` | `143022` |
| `{yyyyMMdd_HHmmss}` | `20260403_143022` |
| `{yyyy}` / `{MM}` / `{dd}` | 年・月・日 単独 |

```
例: C:\output\report_{yyyyMMdd}.xlsx → C:\output\report_20260403.xlsx
```

出力先ディレクトリが存在しない場合は自動作成する。

## ジョブ機能

### 概要

複数のパイプラインを指定した順序で順次実行する。ジョブ自体も `.job.json` ファイルとして保存・共有できる。

### Job モデル

| プロパティ | 説明 |
|---|---|
| Id | Guid（自動生成）|
| Name | ジョブ名 |
| IsEnabled | 有効/無効 |
| Steps | JobStep のリスト（Order 昇順で実行）|
| LastRunTime / LastRunSuccess / LastRunMessage | 最終実行情報 |
| FilePath | 保存先ファイルパス（JSON に含めない）|

### JobStep モデル

| プロパティ | 説明 |
|---|---|
| Order | 実行順序（0 始まり）|
| Name | ステップ表示名 |
| PipelineFilePath | 実行するパイプラインの JSON ファイルパス |
| ContinueOnError | `true`：このステップでエラーが発生しても次のステップへ続行 |

### ジョブファイル形式

```json
{
  "Id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "Name": "日次バッチ",
  "IsEnabled": true,
  "Steps": [
    {
      "Id": "...",
      "Order": 0,
      "Name": "マスタ更新",
      "PipelineFilePath": "C:\\pipelines\\master_update.json",
      "ContinueOnError": false
    },
    {
      "Id": "...",
      "Order": 1,
      "Name": "集計処理",
      "PipelineFilePath": "C:\\pipelines\\aggregate.json",
      "ContinueOnError": true
    }
  ]
}
```

### ジョブレジストリ

`JobRegistry`（シングルトン）が最近使用したジョブ一覧を `jobs.json` に保持。ジョブマネージャーダイアログの左パネルに表示される。

## スケジュール機能

### ScheduleEntry モデル

| プロパティ | 説明 |
|---|---|
| Target | `Pipeline`（パイプラインファイル直接実行）/ `Job`（ジョブファイル実行）|
| PipelineFilePath | Target = Pipeline の場合に使用 |
| JobFilePath | Target = Job の場合に使用 |
| Mode | `InApp`（アプリ起動中のみ）/ `TaskScheduler`（Windows タスクスケジューラ）|
| Type | `Daily` / `Weekly` / `Hourly` / `Interval` |
| IsEnabled | 有効/無効 |

### スケジュール種別

| 種別 | 説明 | 設定項目 |
|---|---|---|
| Daily | 毎日 指定時刻 | HH:MM |
| Weekly | 毎週 指定曜日・時刻 | 曜日 + HH:MM |
| Hourly | 毎時 指定分 | MM |
| Interval | N 分ごと | 間隔（分）|

### 実行モード

| モード | 説明 |
|---|---|
| インアプリ | アプリが起動している間のみ DispatcherTimer（1分周期）で監視・実行 |
| Windowsタスクスケジューラ | `schtasks.exe` でタスクを登録。アプリ未起動でも OS が定期実行。パイプラインのみ対応 |

スケジューラ Tick は `ScheduleRegistry.CalcLastDueTime()` で直近の実行予定時刻を計算し、`LastRunTime` と比較して未実行であれば実行する。Target が Job の場合は `JobLoader.LoadFromFile()` → `JobExecutor.ExecuteAsync()` を呼び出す。

## 未保存変更の検出

### パイプライン

以下の操作で `IsModified = true` となりタイトルバーに `*` が付く。

- ステップの追加・削除・接続
- ステップ設定ダイアログを閉じた後

**保存時**（`SavePipeline`）、**新規作成**（`NewPipeline`）、**開く**（`LoadPipeline`）、**クリア**（`ClearCanvas`）の前に未保存確認ダイアログを表示。ウィンドウの `×` ボタンでも同様に確認する。

### ジョブ

以下の操作で `_isJobDirty = true` となりダイアログタイトルに `*` が付く。

- ジョブ名・有効フラグの変更
- ステップの追加・削除・移動（↑↓）・適用

**ジョブ切替時**および**ダイアログを閉じる時**に未保存確認を行う。ファイル保存（💾 保存 / 📋 名前を付けて保存）成功後にフラグをクリアする。

> ジョブの「レジストリ保存（右下の保存ボタン）」はファイルへの保存ではないため、dirty フラグをクリアしない。

## UI コンポーネント

### ツールバー

|---|---|
| 📄 新規 | 新しいパイプライン（未保存確認あり）|

登録済みスケジュールを一覧表示。各カードに **有効/無効トグルボタン** を持ち即座に切り替え・保存できる。
