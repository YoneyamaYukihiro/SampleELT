# SampleELT 仕様書

## 概要

SampleELT は、Pentaho Spoon に類似したビジュアル ETL（Extract / Transform / Load）ツールです。
Oracle・MySQL データベースおよびファイル（Excel / CSV / TXT / TSV）を対象に、データの抽出・変換・出力をドラッグ＆ドロップで設計・実行できます。

## アーキテクチャ

### データフロー

```csharp
Task<List<Dictionary<string, object?>>> ExecuteAsync(
    List<Dictionary<string, object?>> inputData,
    IProgress<string> progress,
    CancellationToken ct)
```

- **inputData** : 前ステップ（単一）の出力行リスト
- **AllInputStreams** : 複数入力時（MergeJoin 用）— `AllInputStreams[0]` = Left、`[1]` = Right
- Output ステップは inputData をそのまま返す（後続ステップへのパススルー）

### 実行エンジン

Kahn's Algorithm でトポロジカルソート → 依存順に非同期実行。
各ステップ実行前に `AllInputStreams` をセットする。

### DB 接続設定

`ConnectionRegistry`（シングルトン）が全接続設定を保持し `connections.json` に永続化。
各ステップは `Settings["ConnectionId"]` (Guid 文字列) で接続を参照する。

Oracle 接続は個別フィールド（ホスト・ポート・サービス名・ユーザー・パスワード）で設定し、Easy Connect 形式（`host:port/service`）に自動変換される。

MySQL 接続は `AllowUserVariables=true` を自動付加するため、SQL 内の `@変数` をユーザー変数として扱える。

### スケジュール機能

`ScheduleRegistry`（シングルトン）がスケジュール設定を保持し `schedules.json` に永続化。
右パネルのスケジュール一覧から有効/無効の切り替えが可能。

## ステップ仕様

### Input グループ

#### DB Input（統合 DB 入力）

接続設定の DB 種別（Oracle / MySQL）を自動判別して適切なドライバを使用する。

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID（ConnectionRegistry から選択）|
| SQL | 実行する SELECT 文 |
| ExecuteEachRow | `true`：前ステップの各行を `?` パラメータとして渡し行ごとに実行 |

**Lookup パラメータ（`ExecuteEachRow=false` 時）**
SQL に `?` が含まれ前ステップに接続されている場合、前ステップの最初の行の値を `?` に順番に当てはめて1回実行。Oracle では `?` は自動的に `:_p0`, `:_p1`... に変換される。

**MySQL ユーザー変数対応**
MySQLInputStep は `AllowUserVariables=true` を接続文字列に自動付加するため、SQL 内の `@変数名` をユーザー変数として利用できる。

#### Oracle Input / MySQL Input（レガシー個別ステップ）

DB Input と同等の機能。既存のパイプライン互換性のために保持。

#### Excel Input

| 設定項目 | 説明 |
|---|---|
| FilePath | Excel ファイルパス (.xlsx) |
| SheetName | 読み込むシート名 |
| HasHeader | 1行目をヘッダーとして扱う（デフォルト: true）|

---

### Output グループ

#### DB Output（統合 DB 出力）

接続設定の DB 種別（Oracle / MySQL）を自動判別して適切なドライバを使用する。

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| TableName | 出力先テーブル名 |
| Mode | INSERT / UPSERT |
| CommitSize | N 件ごとにコミット（デフォルト: 100、0 = 最後に一括コミット）|

#### File Output (ExcelOutput ステップ)

| 設定項目 | 説明 |
|---|---|
| FilePath | 出力ファイルパス |
| Format | `Excel` / `CSV` / `TSV` / `TXT` |
| SheetName | シート名（Excel のみ）|
| Delimiter | 区切り文字（CSV/TXT: `,`  TSV: `\t` ）|
| Encoding | `UTF-8` / `UTF-8 BOM` / `Shift-JIS` |
| IncludeHeader | ヘッダー行を出力する（デフォルト: true）|

出力先ディレクトリが存在しない場合は自動作成する。

---

### Transform グループ

#### Filter

| 設定項目 | 説明 |
|---|---|
| FieldName | フィルタ対象フィールド名 |
| Operator | equals / notEquals / contains / greaterThan / lessThan / isNull / isNotNull |
| Value | 比較値（固定値）|
| RightField | 比較対象フィールド名（指定時はフィールド間比較、Value より優先）|

`RightField` を設定すると、`FieldName` と `RightField` のフィールド値を直接比較する（DateTime 対応）。

#### Calculation

| 設定項目 | 説明 |
|---|---|
| OutputFieldName | 出力フィールド名 |
| ExpressionType | add / subtract / multiply / divide / concat / constant / **dateDiffMinutes** |
| Field1 / Field2 | 演算対象フィールド名 |
| Constant | 固定値（constant 式の場合）|

**dateDiffMinutes**: `Field1 - Field2` の差分を分単位で返す。両フィールドが DateTime 型の場合は `TotalMinutes` を使用し、数値の場合は `(Field1 - Field2) * 24 * 60` を計算する。

#### Select Values

| 設定項目 | 説明 |
|---|---|
| FieldMappings | 1行1フィールド。`元名=新名` または `フィールド名` のみ。記述したフィールドのみ出力 |

---

### DB Operation グループ

#### DB Delete

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| TableName | 削除対象テーブル名 |
| KeyFields | キーフィールド名（カンマ区切り）|
| CommitSize | N 件ごとにコミット（デフォルト: 100）|

#### Insert / Update（UPSERT）

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| TableName | 対象テーブル名 |
| KeyFields | 検索キーフィールド名 |
| UpdateFields | 更新フィールド名（空 = キー以外すべて）|
| CommitSize | N 件ごとにコミット（デフォルト: 100）|

Oracle: `MERGE INTO`、MySQL: `ON DUPLICATE KEY UPDATE` を使用。

#### DB Update（UPDATE のみ）

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| TableName | 対象テーブル名 |
| KeyFields | 検索キーフィールド名 |
| UpdateFields | 更新フィールド名（空 = キー以外すべて）|
| CommitSize | N 件ごとにコミット（デフォルト: 100）|

INSERT しない。一致しない場合はスキップ。

#### Exec SQL

| 設定項目 | 説明 |
|---|---|
| ConnectionId | 接続設定 ID |
| SQL | 実行する SQL |
| ExecuteEachRow | `true`：前ステップの各行を ? パラメータとして渡し行ごとに実行 |

---

### Utility グループ

#### Dummy

入力データをそのまま通過させる。分岐の終端などに使用。

#### Generate Rows

| 設定項目 | 説明 |
|---|---|
| RowCount | 生成行数 |
| Fields | フィールド定義。`フィールド名=デフォルト値` を改行区切り |

#### Merge Join

| 設定項目 | 説明 |
|---|---|
| JoinType | INNER / LEFT OUTER / RIGHT OUTER / FULL OUTER |
| KeyFields | 結合キーフィールド名（カンマ区切り）|

最初に接続されたストリームが Left、2番目が Right。

#### Set Variable

| 設定項目 | 説明 |
|---|---|
| Fields | 変数定義（`変数名=式`、改行区切り）|
| DateFormat | 日付書式（デフォルト: `yyyy/MM/dd`）|

**使用可能な式**

| 式 | 説明 |
|---|---|
| `TODAY` | 本日の日付 |
| `YESTERDAY` | 昨日の日付 |
| `TODAY+N` / `TODAY-N` | 本日 ± N 日 |
| `MONTH_START` | 当月1日 |
| `MONTH_END` | 当月末日 |
| `YEAR_START` | 当年1月1日 |
| `YEAR_END` | 当年12月31日 |
| 固定値 | そのまま文字列として出力 |

常に1行のデータを出力する（後続ステップへの変数渡しに使用）。

#### JavaScript

| 設定項目 | 説明 |
|---|---|
| Script | JavaScript コード（Jint 3.x で実行）|
| RunPerRow | `true`：入力の各行に対して実行 |

**JS 環境**
- 入力フィールド値を変数として参照可能
- `$out.フィールド名 = 値` で出力フィールドを設定
- 入力フィールドのうち `$out` に設定しないものはそのまま引き継がれる

```javascript
// 使用例
var today = new Date();
$out.result_date = today.getFullYear() + '/'
    + (today.getMonth() + 1) + '/'
    + today.getDate();
```

## パイプライン保存形式

`.json` 拡張子で保存（JSONシリアライズ）。プロパティ名は PascalCase。

```json
{
  "Name": "サンプルパイプライン",
  "Steps": [
    {
      "Id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "Name": "DB Input",
      "StepType": "DBInput",
      "CanvasX": 100,
      "CanvasY": 150,
      "NodeWidth": 150.0,
      "NodeHeight": 70.0,
      "Settings": {
        "ConnectionId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
        "SQL": "SELECT * FROM orders",
        "ExecuteEachRow": "false"
      }
    }
  ],
  "Connections": [
    {
      "Id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
      "SourceStepId": "...",
      "TargetStepId": "..."
    }
  ]
}
```

読み込み時はプロパティ名の大文字・小文字を区別しない（`PropertyNameCaseInsensitive = true`）。

## UI コンポーネント

### パレット グループ

パレットのボタンは **ダブルクリック** でキャンバスにステップを追加する。

| グループ | 色 | ステップ |
|---|---|---|
| Input | 青 | 🔌 DB Input / 📂 File Input |
| Output | 緑 | 🔌 DB Output / 📄 File Output / 🔄 Insert/Update / ✏️ DB Update |
| Transform | オレンジ | 🔍 Filter / 🧮 Calculation / 📋 Select Values |
| DB Operation | ピンク | 🗑 DB Delete / ⚡ Exec SQL |
| Utility | 紫 | 🔢 Generate Rows / 🔗 Merge Join / 📅 Set Variable / ⬜ Dummy |

### ステップノードカード

- **ステップ名**（太字）・**種別ラベル**・**接続設定名**（青字、DB接続ありの場合のみ表示）を表示
- 右端の **青い ▶** が出力ポート（ドラッグして接続）
- 接続先として有効な場合は **緑ボーダー**でハイライト
- 選択時は **青ボーダー**
- 右下の **リサイズグリップ**（斜め矢印カーソル）をドラッグしてノードサイズを変更可能（最小幅120px・最小高さ50px）

### 接続線・矢印

接続線はソースノードの**右端中央**（出力ポート）からターゲットノードの**左端中央**（入力ポート）へ描画される。
矢印はターゲットノード左端に表示され、データの流れ方向を示す。

### ログパネル

- リアルタイムで実行状況を表示（自動スクロール）
- **📋 コピー** ボタンで全ログをクリップボードにコピー
- **クリア** ボタンでログを消去

### 右パネル（スケジュール一覧）

登録済みスケジュールを一覧表示。各スケジュールカードに **有効/無効トグルボタン** を持ち、即座に切り替え・保存できる。
