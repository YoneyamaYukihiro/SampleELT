# SampleELT 使い方ガイド

## 起動方法

1. Visual Studio 2022 で `SampleELT.csproj` を開く
2. NuGet パッケージの復元を待つ（初回は数分かかる場合あり）
3. **F5** キーでビルド＆起動（または Ctrl+F5）

---

## 基本的な操作フロー

```
① DB接続登録  →  ② ステップ追加  →  ③ 設定入力  →  ④ ステップ接続  →  ⑤ 実行
```

---

## ① DB 接続設定の登録

メニュー **[ツール] → [DB 接続設定]** を開き、Oracle / MySQL の接続を登録します。  
登録した接続は各 Input / Output ステップのダイアログで共通して選択できます。

接続設定は `connections.json`（実行ファイルと同じフォルダ）に自動保存されます。

---

## ② ステップの追加

左側の **ステップパレット** からボタンを **ダブルクリック** すると、キャンバスに追加されます。

### Input グループ（青）

| ステップ | 説明 |
|---|---|
| 🔶 Oracle Input | Oracle DB から SELECT でデータ取得 |
| 🐬 MySQL Input | MySQL DB から SELECT でデータ取得 |
| 📗 Excel Input | Excel (.xlsx) ファイルからデータ読み込み |

### Output グループ（緑）

| ステップ | 説明 |
|---|---|
| 🔶 Oracle Output | Oracle テーブルへ INSERT / UPSERT |
| 🐬 MySQL Output | MySQL テーブルへ INSERT / UPSERT |
| 📄 File Output | Excel / CSV / TSV / TXT ファイルへ書き出し |

### Transform グループ（オレンジ）

| ステップ | 説明 |
|---|---|
| 🔍 Filter | 条件に一致する行のみ通過させる |
| 🧮 Calculation | 演算結果を新フィールドとして追加 |
| 📋 Select Values | フィールドを選択・リネーム（不要なフィールドを除去） |

### DB Operation グループ（ピンク）

| ステップ | 説明 |
|---|---|
| 🗑 DB Delete | 入力データのキーでDB行を削除 |
| 🔄 Insert/Update | キー一致→UPDATE、なければ INSERT（UPSERT）|
| ✏️ DB Update | キー一致→UPDATE のみ（INSERT しない）|
| ⚡ Exec SQL | 任意の SQL を実行 |

### Utility グループ（紫）

| ステップ | 説明 |
|---|---|
| 🔢 Generate Rows | 固定値の行を N 件生成 |
| 🔗 Merge Join | 2つの入力ストリームをキーで結合 |
| 📜 JavaScript | JavaScript でデータ変換 |
| ⬜ Dummy | 何もしないパススルー（分岐終端）|

---

## ③ ステップの設定

ステップノードを **ダブルクリック** または **右クリック → 設定** で設定ダイアログが開きます。

### Oracle Input / MySQL Input の設定

1. ドロップダウンから接続を選択（⚙ 接続設定管理 で新規追加も可）
2. SQL クエリを入力
3. **[🔗 接続テスト]** で疎通確認

**行ごとに実行（execute_each_row）**  
前ステップの各行の値を `?` パラメータとして渡してクエリを繰り返し実行します。  
SQL に `?` を含め、チェックボックスをオンにしてください。

```sql
-- 例: 前ステップの WP_ID をパラメータとして検索
SELECT * FROM process_log WHERE WP_ID = ?
```

**Lookup パラメータ（チェックオフ時）**  
前ステップに接続されていて SQL に `?` が含まれる場合、前ステップの**最初の行**の値を `?` に順番に当てはめて1回実行します。

```sql
-- 例: 前ステップから start_date, end_date を受け取る
SELECT * FROM real_process_time
WHERE PROC_END >= ? AND PROC_END < ?
```

### File Output の設定

| 項目 | 説明 |
|---|---|
| 出力フォーマット | Excel (.xlsx) / CSV (.csv) / TSV (.tsv) / テキスト (.txt) |
| 出力ファイルパス | [参照...] ボタンで選択（または直接入力）|
| シート名 | Excel のみ設定（デフォルト: Sheet1）|
| 区切り文字 | CSV/TXT: カンマ / タブ / パイプ / カスタム |
| 文字エンコーディング | UTF-8 / UTF-8 BOM / Shift-JIS |
| ヘッダー行 | 1行目にフィールド名を出力するか |

> ⚠ 出力先フォルダが存在しない場合は自動作成されます。

### Select Values の設定

1行1フィールドで記述します。

```
# そのまま出力（フィールド名のみ）
emp_id
emp_name

# リネームして出力（元名=新名）
salary=SALARY_JPY
dept_id=DEPARTMENT_ID
```

記述したフィールドのみ出力されます（未記述のフィールドは削除）。

### DB Delete の設定

| 項目 | 説明 |
|---|---|
| DB 接続 | 接続を選択 |
| テーブル名 | 削除対象テーブル |
| キーフィールド | 検索キー（例: `id` または `dept_id,emp_id`）|

入力データの各行をキーとして `DELETE WHERE キー = ?` を実行します。

### Insert / Update の設定

| 項目 | 説明 |
|---|---|
| キーフィールド | 検索キー（カンマ区切り）|
| 更新フィールド | 一致時に更新するフィールド（空 = キー以外すべて）|

Oracle は `MERGE INTO`、MySQL は `ON DUPLICATE KEY UPDATE` を使用。

### DB Update の設定

Insert/Update と同じ設定ですが、キー一致行のみ UPDATE します（INSERT しません）。

### Merge Join の設定

| 項目 | 説明 |
|---|---|
| 結合タイプ | INNER / LEFT OUTER / RIGHT OUTER / FULL OUTER |
| キーフィールド | 結合キー（カンマ区切り） |

> ⚠ **最初に接続したストリームが Left、2番目が Right** になります。

### Generate Rows の設定

```
# フィールド定義（フィールド名=デフォルト値）
start_date=
end_date=
counter=1
```

### JavaScript の設定

入力フィールドの値は変数として参照できます。  
出力フィールドは `$out` オブジェクトに設定してください。

```javascript
// 例: 昨日・今日の日付を計算してフィールドに設定
var today = new Date();
var yesterday = new Date();
yesterday.setDate(yesterday.getDate() - 1);

$out.start_date = yesterday.getFullYear() + '/'
    + (yesterday.getMonth() + 1) + '/'
    + yesterday.getDate();

$out.end_date = today.getFullYear() + '/'
    + (today.getMonth() + 1) + '/'
    + today.getDate();
```

`$out` に設定しない入力フィールドはそのまま引き継がれます。

## ④ ステップの接続

各ステップノードの**右端にある青い ▶ ボタン（出力ポート）**からドラッグして、接続先ステップへドロップします。

```
[MySQL Input] ─▶── [Select Values] ─▶── [File Output]
```

### 操作手順
1. 接続元ステップの **右端の青い ▶** にマウスを合わせる（カーソルが十字になる）
2. そのまま**ドラッグ**して接続先ステップへ
3. 接続先ステップが**緑色にハイライト**されたらドロップ → 接続完成
4. キャンセルするには **Escape** キーを押す

### 複数入力（Merge Join）

Merge Join ステップには2本の接続が必要です。

```
[MySQL Input A] ─▶──┐
                     ├─► [Merge Join] ─▶── [File Output]
[MySQL Input B] ─▶──┘
```

最初に接続した方が Left、後から接続した方が Right になります。

## ⑤ パイプラインの実行

ツールバーの **▶ 実行** ボタンをクリック（または F5）。

- ステータスバーにプログレスバーが表示されます
- 下部のログパネルに処理状況がリアルタイムで表示されます
- **■ 停止** ボタンでキャンセル可能です

### ログ表示例

```
[09:15:23] ===== パイプライン実行開始 =====
[09:15:23] [MySQL Input] 実行中...
[09:15:24] MySQL Input: 1,250行 読み込み完了
[09:15:24] [Select Values] 実行中...
[09:15:24] Select Values: 1,250行 変換完了
[09:15:24] [File Output] 実行中...
[09:15:25] File Output: 1,250行 書き込み完了 → C:\output\result.xlsx
[09:15:25] ===== 実行完了 =====
```

## よくある質問

### Q. File Output でファイルが出力されない

- ファイルパスが設定されているか確認してください（ダブルクリックで設定を開く）
- ログパネルのエラーメッセージを確認してください
- 出力先フォルダへの書き込み権限を確認してください
- フォルダが存在しない場合は自動作成されます

### Q. CSV / TXT を日本語で出力したい

エンコーディングを **Shift-JIS** または **UTF-8 BOM** に設定してください。  
Excel で直接開く場合は Shift-JIS が確実です。

### Q. Oracle に接続できない

```
// TNS名方式
Data Source=ORCL;User Id=scott;Password=tiger;

// ホスト直接指定
Data Source=192.168.1.100:1521/ORCL;User Id=scott;Password=tiger;
```

### Q. MySQL の SELECT に ? パラメータを使いたい

前ステップ（例: Generate Rows → JavaScript）からパラメータを渡す場合：  
SQL に `?` を記述し、ダイアログの設定に応じて動作が変わります。

- **行ごとに実行 チェックON** → 前ステップの各行を ? に当てはめて繰り返し実行
- **チェックOFF、前ステップあり** → 前ステップの最初の行の値を1回だけ当てはめて実行

### Q. Merge Join で Left / Right の順序を変えたい

一度接続を削除して、希望の順序で再接続してください。

### Q. JavaScript でエラーが発生する

- Jint 3.x は ES5 準拠です。`let` / `const` の代わりに `var` を使ってください
- 出力は `$out.フィールド名 = 値;` の形式で記述してください
- `console.log()` は使用できません
