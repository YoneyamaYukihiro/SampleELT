# SampleELT 使い方ガイド

## 起動方法

1. Visual Studio 2022 で `SampleELT.csproj` を開く
2. NuGet パッケージの復元を待つ（初回は数分かかる場合あり）
3. **F5** キーでビルド＆起動（または Ctrl+F5）

---

## 基本的な操作フロー

```
① DB接続登録  →  ② ステップ追加  →  ③ 設定入力  →  ④ ステップ接続  →  ⑤ 実行
```

---

## ① DB 接続設定の登録

メニュー **[ツール] → [DB 接続設定]** を開き、利用したい DB の接続を登録します。  
登録した接続は各 Input / Output ステップのダイアログで共通して選択できます。

### 対応 DB と既定値

| DB 種別 | アイコン | 既定ポート | 主な入力項目 |
|---|---|---|---|
| Oracle | 🔶 | 1521 | サーバー / ポート / サービス名 / ユーザー / パスワード |
| MySQL | 🐬 | 3306 | サーバー / ポート / DB名 / ユーザー / パスワード |
| PostgreSQL | 🐘 | 5432 | サーバー / ポート / DB名 / ユーザー / パスワード |
| SQL Server | 🟦 | 1433 | サーバー / ポート / DB名 / Windows認証 or ユーザー/パスワード |
| MariaDB | 🦭 | 3306 | サーバー / ポート / DB名 / ユーザー / パスワード（MySQL と互換） |
| SQLite | 🪶 | (なし) | データベースファイル（[参照…] で選択／ファイルが無ければ自動生成）|

各接続にはアイコン付きの表示名（例: `🐘 本番PostgreSQL`）が一覧に表示されます。  
**[🔗 接続テスト]** ボタンで疎通確認ができます。  
接続設定は `connections.json`（実行ファイルと同じフォルダ）に自動保存されます。

> 💡 SQL Server で TLS 証明書エラーが出る場合は、ビルダーで `TrustServerCertificate=true` を自動付与しているため通常は問題ありませんが、運用時はサーバ証明書の検証ポリシーをご確認ください。

## ② ステップの追加

左側の **ステップパレット** からボタンを **ダブルクリック** すると、キャンバスに追加されます。

### Input グループ（青）

| ステップ | 説明 |
|---|---|
| 🗄 DB Input | 主要 RDBMS（Oracle / MySQL / PostgreSQL / SQL Server / MariaDB / SQLite）から SELECT でデータ取得 |
| 📗 File Input | Excel (.xlsx) ファイルからデータ読み込み |

### Output グループ（緑）

| ステップ | 説明 |
|---|---|
| 💾 DB Output | 主要 RDBMS のテーブルへ INSERT / UPSERT |
| 📄 File Output | Excel / CSV / TSV / TXT ファイルへ書き出し（日時トークン対応）|

### Transform グループ（オレンジ）

| ステップ | 説明 |
|---|---|
| 🔍 Filter | 条件に一致する行のみ通過させる |
| 🧮 Calculation | 演算結果を新フィールドとして追加 |
| 📋 Select Values | フィールドを選択・リネーム（不要なフィールドを除去） |
| 📌 Set Variable | TODAY / YESTERDAY などの日付変数を生成 |

### DB Operation グループ（ピンク）

| ステップ | 説明 |
|---|---|
| 🗑 DB Delete | 入力データのキーでDB行を削除 |
| 🔄 Insert/Update | キー一致→UPDATE、なければ INSERT（UPSERT）|
| ✏️ DB Update | キー一致→UPDATE のみ（INSERT しない）|
| ⚡ Exec SQL | 任意の SQL を実行 |

> いずれも対応 6 DB（Oracle / MySQL / PostgreSQL / SQL Server / MariaDB / SQLite）で動作。UPSERT 構文は接続先 DB に応じて `MERGE` / `ON DUPLICATE KEY UPDATE` / `ON CONFLICT DO UPDATE` を自動選択します。

### Utility グループ（紫）

| ステップ | 説明 |
|---|---|
| 🔗 Merge Join | 2つの入力ストリームをキーで結合 |
| ⬜ Dummy | 何もしないパススルー（分岐終端）|

## ③ ステップの設定

ステップノードを **ダブルクリック** または **右クリック → 設定** で設定ダイアログが開きます。

### DB Input の設定

1. ドロップダウンから **接続** を選択（全 6 DB 共通の UI）
2. **[↓ 一覧取得]** でテーブル一覧をプルダウンに読み込み（任意）
3. **SQL クエリ** を入力（または **[SQL 自動生成]** で `SELECT * FROM table` を生成）
4. **[🔗 接続テスト]** で疎通確認、**[👁 プレビュー]** で先頭 N 行を確認

**行ごとに実行（execute_each_row）**  
前ステップの各行の値を `?` パラメータとして渡してクエリを繰り返し実行します。

```sql
-- 例: 前ステップの WP_ID をパラメータとして検索
SELECT * FROM process_log WHERE WP_ID = ?
```

**Lookup パラメータ（チェックオフ時）**  
前ステップに接続されていて SQL に `?` が含まれる場合、前ステップの**最初の行**の値を `?` に順番に当てはめて1回実行します。

**名前付きバインド `:{fieldname}`**  
前ステップ行のフィールドを名前で参照できます。DB ごとに自動変換されます。

```sql
SELECT * FROM orders WHERE customer_id = :{cust_id} AND status = :{status}
```

### DB Output の設定

| 項目 | 説明 |
|---|---|
| DB 接続 | 接続を選択（全 6 DB 共通） |
| テーブル名 | 書き込み先テーブル |
| キーフィールド | UPSERT 時の検索キー（カンマ区切り） |
| 更新フィールド | 一致時に更新するフィールド（空 = キー以外すべて） |
| コミットサイズ | N 件ごとに中間コミット（デフォルト: 100、0 = 一括コミット） |

### File Output の設定

| 項目 | 説明 |
|---|---|
| 出力フォーマット | Excel (.xlsx) / CSV (.csv) / TSV (.tsv) / テキスト (.txt) |
| 出力ファイルパス | [参照...] ボタンで選択（または直接入力）|
| シート名 | Excel のみ設定（デフォルト: Sheet1）|
| 区切り文字 | CSV/TXT: カンマ / タブ / パイプ / カスタム |
| 文字エンコーディング | UTF-8 / UTF-8 BOM / Shift-JIS |
| ヘッダー行 | 1行目にフィールド名を出力するか |

#### ファイル名に日付・時刻を埋め込む

ファイルパスに `{フォーマット}` の形式でトークンを記述すると、実行時の日時に自動展開されます。  
ダイアログ内のボタンをクリックするとキャレット位置にトークンが挿入され、プレビューが表示されます。

| ボタン（トークン） | 展開例 |
|---|---|
| `{yyyyMMdd}` | `20260403` |
| `{yyyy-MM-dd}` | `2026-04-03` |
| `{HHmmss}` | `143022` |
| `{HH-mm-ss}` | `14-30-22` |
| `{yyyyMMdd_HHmmss}` | `20260403_143022` |
| `{yyyy}` | `2026` |
| `{MM}` | `04` |
| `{dd}` | `03` |

```
例: C:\output\report_{yyyyMMdd}.xlsx
　　→ 実行時: C:\output\report_20260403.xlsx
```

C# の標準日時フォーマット文字列（`yyyy`、`MM`、`dd`、`HH`、`mm`、`ss` など）はすべて使用可能です。

> ⚠ 出力先フォルダが存在しない場合は自動作成されます。

### Select Values の設定

```
# そのまま出力（フィールド名のみ）
emp_id
emp_name

# リネームして出力（元名=新名）
salary=SALARY_JPY
dept_id=DEPARTMENT_ID
```

記述したフィールドのみ出力されます（未記述のフィールドは削除）。

### その他のステップ設定

| ステップ | 主な設定 |
|---|---|
| DB Delete | 接続 / テーブル名 / キーフィールド / コミットサイズ |
| Insert/Update | 接続 / テーブル名 / キーフィールド / 更新フィールド / コミットサイズ |
| DB Update | Insert/Update と同じ（UPDATE のみ、INSERT しない）|
| Exec SQL | 接続 / SQL / 行ごとに実行 |
| Merge Join | 結合タイプ（INNER/LEFT/RIGHT/FULL）/ キーフィールド |
| Set Variable | 変数定義（`変数名=式`、TODAY / YESTERDAY / TODAY+N など）|

## ④ ステップの接続

各ステップノードの**右端にある青い丸（出力ポート）**からドラッグして、接続先ステップへドロップします。

```
[DB Input] ──► [Select Values] ──► [File Output]
```

1. 接続元ステップの **右端の青い丸** にマウスを合わせる（カーソルが十字になる）
2. そのまま**ドラッグ**して接続先ステップへ
3. 接続先が**緑色にハイライト**されたらドロップ → 接続完成
4. キャンセルするには **Escape** キーを押す

## パイプラインの保存・読み込み

| 操作 | 方法 |
|---|---|
| 保存 | **[ファイル] → [保存]** または Ctrl+S |
| 開く | **[ファイル] → [開く]** または Ctrl+O |
| 新規作成 | **[ファイル] → [新規]** または Ctrl+N |

`.json` 形式でステップの設定・配置・接続・ノードサイズがすべて保存されます。

### 未保存の変更

パイプラインに未保存の変更がある場合、タイトルバーに `*` が付きます。  
新規・開く・クリア・ウィンドウを閉じる際に確認ダイアログが表示されます。

## ジョブ機能

ジョブは **複数のパイプラインを指定した順番で順次実行** する機能です。  
ツールバーの **🔗 ジョブ** ボタン（または [ツール] → [ジョブ管理]）で管理ダイアログを開きます。

### ジョブの作成

1. ツールバーの「**📄 新規**」ボタンでジョブを新規作成
2. ジョブ名・ログ出力モードを設定
3. 「**＋ 追加**」でパイプラインステップを追加
4. 各ステップに **パイプラインファイルを指定**（[参照...] ボタン）
5. 必要に応じてステップ名・エラー時の動作を設定
6. **↑↓ ボタン** で実行順序を変更

| 項目 | 説明 |
|---|---|
| ステップ名 | ジョブ内での表示名（任意）|
| ファイル | 実行するパイプラインの .json ファイルパス |
| エラー時も続行する | チェックON: このステップでエラーが発生しても次のステップへ進む |

### ジョブのファイル保存・読み込み

ジョブはパイプラインと同様に **ファイル（.job.json）** として保存・共有できます。

| ボタン | 機能 |
|---|---|
| 📄 新規 | 新しいジョブを作成 |
| 📂 開く | .job.json ファイルを開く |
| 💾 保存 | 現在のジョブを既存のファイルパスに上書き保存 |
| 📋 名前を付けて保存 | 新しいファイルパスに保存 |
| ▶ 実行 | 開いているジョブをその場で実行（未保存なら保存確認） |
| ■ 停止 | 実行中のジョブをキャンセル |

> ファイルに未保存の変更がある場合、タイトルに `*` が付きます。  
> 別のジョブへ切替時またはダイアログを閉じる際に確認ダイアログが表示されます。

### ジョブの手動実行

ツールバーの「**▶ 実行**」でジョブを UI から直接実行できます。

- 未保存の変更があると「保存して実行 / 保存せず実行 / 中止」を確認します
- 実行中は他のボタンが無効化され、進捗ステータスがダイアログ下部に表示されます
- 全てのログメッセージは **メインウィンドウのログパネル** にも出力され、ダイアログを閉じた後に履歴を確認できます
- ジョブ実行中はダイアログを閉じられません（先に「■ 停止」を押す）

## スケジュール実行

ツールバーの **⏰ スケジュール** ボタン（または右パネルから）でスケジュール管理ダイアログを開きます。

### スケジュールの設定項目

| 項目 | 説明 |
|---|---|
| スケジュール名 | 識別用の名前 |
| **アプリ起動中のみ監視する** | アプリ内スケジューラで監視するかどうか（既定: OFF）。**Windows タスクスケジューラ モードでは Windows タスク側の有効/無効が実際の実行を制御** |
| **実行ファイル** | パイプライン（`.json`）またはジョブ（`.job.json`）を [参照...] で選択。種類は拡張子・内容から自動判定し「種類: パイプライン / ジョブ」と表示 |
| **実行モード** | **インアプリ** または **Windows タスクスケジューラ** |
| スケジュール種類 | 毎日 / 毎週 / 毎時 / インターバル |

### 実行モード

| モード | 説明 | 対応対象 |
|---|---|---|
| インアプリ | アプリが起動している間のみ監視・実行 | パイプライン・ジョブ |
| Windowsタスクスケジューラ | アプリ未起動でも OS が定期実行（管理者権限が必要な場合あり）| **パイプライン・ジョブ両対応** |

### スケジュール種別

| 種別 | 設定項目 |
|---|---|
| 毎日（指定時刻） | 実行時刻（HH:MM）|
| 毎週（指定曜日・時刻） | 曜日 + 実行時刻 |
| 毎時（指定分） | 毎時 XX 分（00〜59）|
| インターバル（N分ごと） | 実行間隔（分）|

### 右パネルのスケジュール一覧

メインウィンドウ右パネルに登録済みスケジュールが一覧表示されます。  
各カードの **有効/無効トグル** で即座に切り替えができます（インアプリ監視のオン／オフ）。Windows タスクスケジューラ モードのスケジュールはトグルでアプリ内監視を切っても Windows タスク側で実行され続けます。

## コマンドライン (CLI) モード

`SampleELT.exe` は GUI 起動以外に、コマンドライン引数を渡してヘッドレスで動作させることができます。Windows タスクスケジューラやバッチファイルから呼び出す場合に使用します。

| 引数 | 用途 |
|---|---|
| `--run <pipeline.json>` | 指定したパイプライン JSON を 1 回だけ実行（GUI を表示しない） |
| `--run-job <job.job.json>` | 指定したジョブを実行（複数パイプラインを順次） |
| `--convert-ktr <input.ktr> [<output.json>]` | Pentaho KTR ファイルを SampleELT JSON に変換 |

### 実行例

```bat
:: パイプラインをコマンドラインから実行
SampleELT.exe --run "C:\pipelines\daily_load.json"

:: ジョブを実行 (複数パイプラインを順次)
SampleELT.exe --run-job "C:\jobs\nightly.job.json"

:: KTR を変換
SampleELT.exe --convert-ktr "C:\old\transform.ktr" "C:\new\transform.json"
```

### ログ出力

`--run` / `--run-job` で実行したログは、対象ファイルと同じフォルダに  
`<ファイル名>_<yyyyMMdd_HHmmss>.log` として保存できます。  
標準出力にも同じ内容が出力されるため、コマンドプロンプトから起動した場合はリアルタイムに進捗を確認できます。

ログファイル出力はパイプライン／ジョブごとに 3 モードから選べます（既定: エラー時のみ）。

| モード | 動作 |
|---|---|
| エラー時のみ (既定) | 例外発生時だけ `.log` を書き出す。成功時はファイルを残さない |
| 常に出力 | 成功・失敗を問わず常に `.log` を書き出す |
| 出力しない | エラーが起きても `.log` を書き出さない |

UI では、メインウィンドウ ツールバー右端の「ログ:」セレクタ（パイプライン用）と、  
ジョブ管理ダイアログの「ログ出力モード」セレクタ（ジョブ用）から切り替えます。  
設定はパイプライン／ジョブの JSON ファイルに保存されます。

### タスクスケジューラ登録時の動作

スケジュール管理ダイアログで「Windows タスクスケジューラ」モードを選んで保存すると、内部的に上記コマンドラインがタスクスケジューラに登録されます。

| Target | 登録されるコマンド |
|---|---|
| パイプライン | `SampleELT.exe --run "<選択したパイプラインファイル>"` |
| ジョブ | `SampleELT.exe --run-job "<選択したジョブファイル>"` |

## よくある質問

### Q. File Output でファイルが出力されない

- ファイルパスが設定されているか確認してください
- ログパネルのエラーメッセージを確認してください
- 出力先フォルダへの書き込み権限を確認してください（フォルダが存在しない場合は自動作成されます）

### Q. ファイル名に今日の日付を付けたい

File Output の設定ダイアログで `{yyyyMMdd}` トークンをファイル名に含めてください。

```
C:\output\result_{yyyyMMdd}.xlsx  →  C:\output\result_20260403.xlsx
```

ダイアログ内のボタンをクリックするとカーソル位置にトークンが挿入され、プレビューも表示されます。

### Q. 複数のパイプラインを順番に実行したい

**ジョブ機能** を使用してください。  
🔗 ジョブ ボタンからジョブ管理ダイアログを開き、実行したいパイプラインを順番に登録します。

### Q. アプリを起動しなくてもスケジュール実行したい

スケジュール管理ダイアログで **実行モード: Windows タスクスケジューラ** を選択し保存してください。  
保存時に `schtasks.exe` でタスクが登録され、アプリ未起動でも定期実行されます。  
※ パイプライン・ジョブのどちらも対応しています。  
※ 管理者権限が必要な場合があります。  
※ 既定では「ユーザーがログオンしている時のみ実行」で登録されます。未ログオン時にも動作させたい場合は、登録後に Windows タスクスケジューラ GUI で「ユーザーがログオンしているかどうかにかかわらず実行する」へ手動変更してください。

### Q. CSV / TXT を日本語で出力したい

エンコーディングを **Shift-JIS** または **UTF-8 BOM** に設定してください。  
Excel で直接開く場合は Shift-JIS が確実です。

### Q. Oracle に接続できない

DB 接続設定ダイアログの各フィールド（ホスト / ポート / サービス名 / ユーザー / パスワード）を個別に入力してください。  
内部的に `host:port/service` 形式に自動変換されます。

### Q. PostgreSQL / SQL Server / MariaDB / SQLite に接続するには

**[ツール] → [DB 接続設定]** ダイアログで「DB 種別」プルダウンから選択し、各 DB 用のフィールドを入力してください。

- **PostgreSQL**: サーバー / ポート（5432） / DB名 / ユーザー / パスワード
- **SQL Server**: サーバー / ポート（1433） / DB名 + Windows 認証チェック（または ユーザー / パスワード）
- **MariaDB**: 入力欄は MySQL と同一（ドライバを共用）
- **SQLite**: 「データベースファイル」に `.db` / `.sqlite` ファイルを指定（[参照…] ボタン）。ファイルが無ければ接続時に自動生成

### Q. SQLite ファイルが作成されない

ファイルパスのフォルダが存在しないと作成に失敗します。出力先フォルダがあるか確認してください。  
また、ファイルパスは絶対パス推奨です。相対パスはアプリの実行ディレクトリ起点になります。

### Q. コミットサイズはどう設定すればよいか

デフォルト（100件）で問題ない場合がほとんどです。  
大量データで速度を優先する場合は大きくし（500〜1000）、  
メモリを節約したい場合は小さく（10〜50）設定してください。  
`0` を指定すると全件処理後に一括コミットします。
