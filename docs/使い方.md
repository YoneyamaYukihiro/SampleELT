# SampleELT 使い方ガイド

## 起動方法

1. Visual Studio 2022 で `SampleELT.csproj` を開く
2. NuGet パッケージの復元を待つ
3. **F5** キーでビルド＆起動（または「デバッグなしで開始」Ctrl+F5）

---

## 基本的な操作フロー

```
① ステップを追加  →  ② 設定を入力  →  ③ ステップを接続  →  ④ 実行
```

---

## ① ステップの追加

左側の **ステップパレット** から追加したいステップのボタンをクリックします。

| グループ | ステップ | 説明 |
|---|---|---|
| Input | 🔶 Oracle Input | Oracle DB からデータを取得 |
| Input | 🐬 MySQL Input | MySQL DB からデータを取得 |
| Input | 📗 Excel Input | Excel ファイルからデータを読み込み |
| Output | 🔶 Oracle Output | Oracle DB テーブルへ書き込み |
| Output | 🐬 MySQL Output | MySQL DB テーブルへ書き込み |
| Output | 📗 Excel Output | Excel ファイルへ書き出し |
| Transform | 🔍 Filter | 条件に一致する行のみ通過させる |
| Transform | 🧮 Calculation | 計算フィールドを追加する |

クリックするとキャンバス上に自動配置されます。

---

## ② ステップの設定

ステップノードを **ダブルクリック** するか、**右クリック → 設定** で設定ダイアログを開きます。

### Oracle Input の設定例

```
接続文字列: Data Source=myserver:1521/ORCL;User Id=myuser;Password=mypass;
SQL: SELECT emp_id, emp_name, salary FROM employees WHERE dept_id = 10
```

**[接続テスト]** ボタンで接続確認ができます。

### MySQL Input の設定例

```
サーバー:  localhost
ポート:    3306
DB名:     mydb
ユーザー:  root
パスワード: ****
SQL:      SELECT * FROM orders WHERE status = 'pending'
```

### Excel Input の設定例

```
ファイルパス: C:\data\input.xlsx   ← [参照...] ボタンで選択
シート名:    Sheet1               ← ファイル選択後、自動で候補が表示される
ヘッダー行:  ✅ あり
```

### Filter の設定例

| 設定 | 値 | 意味 |
|---|---|---|
| フィールド名 | salary | salary フィールドを対象に |
| 演算子 | greaterThan | より大きい |
| 値 | 300000 | 300,000 より大きい行のみ通過 |

利用可能な演算子:

| 演算子 | 意味 |
|---|---|
| equals | 等しい |
| notEquals | 等しくない |
| contains | 含む（文字列） |
| greaterThan | より大きい |
| lessThan | より小さい |
| isNull | NULL である |
| isNotNull | NULL でない |

### Calculation の設定例

**例1: 消費税を計算して新フィールドに追加**
```
出力フィールド名: price_with_tax
演算タイプ:      multiply
フィールド1:     price
フィールド2:     1.1
```

**例2: 姓と名を結合**
```
出力フィールド名: full_name
演算タイプ:      concat
フィールド1:     last_name
フィールド2:     first_name
```

**例3: 固定値を設定**
```
出力フィールド名: status
演算タイプ:      constant
定数値:          processed
```

## ③ ステップの接続

各ステップノードの**右端にある青い ▶ ボタン（出力ポート）**からドラッグして、接続先ステップへドロップします。

```
[Oracle Input] ─▶ドラッグ─► [Filter] ─▶ドラッグ─► [Excel Output]
```

### 操作手順
1. 接続元ステップの **右端の青い ▶** にマウスを合わせる（カーソルが十字になる）
2. そのまま**ドラッグ**して接続先ステップへ
3. 接続先ステップが**緑色にハイライト**されたらドロップ → 接続完成

### その他
- 接続をキャンセルするには **Escape** キーを押す
- 複数の後続ステップに接続可能（分岐）
- 接続を削除するには、接続線を右クリック → 削除

---

## ④ パイプラインの実行

メニュー **[実行] → [パイプラインを実行]** またはツールバーの **▶ ボタン** をクリックします。

- 実行中はステータスバーにプログレスバーが表示されます
- 画面下部のログパネルに処理状況がリアルタイムで表示されます
- **■ 停止** ボタンでキャンセル可能です

### ログ表示例

```
[INFO]  パイプライン開始: サンプルパイプライン
[INFO]  Oracle Input: 1,250行 読み込み完了
[INFO]  Filter: 1,250行 → 487行 (フィルタ後)
[INFO]  Calculation: price_with_tax フィールドを追加
[INFO]  Excel Output: 487行 を output.xlsx に書き込み完了
[INFO]  パイプライン完了
```

## パイプラインの保存・読み込み

### 保存

**[File] → [Save]** または **Ctrl+S**

- `.etl` 形式（JSON）で保存されます
- ステップの設定・配置・接続がすべて保存されます

### 読み込み

**[File] → [Open]** または **Ctrl+O**

- `.etl` ファイルを選択して開きます
- キャンバスにステップが復元されます

## よくある質問

### Q. Oracle に接続できない

接続文字列の形式を確認してください。

```
// TNS名方式
Data Source=ORCL;User Id=scott;Password=tiger;

// ホスト直接指定
Data Source=192.168.1.100:1521/ORCL;User Id=scott;Password=tiger;
```

### Q. Excel が読み込めない

- `.xlsx` 形式のみ対応しています（`.xls` 非対応）
- ファイルが他のアプリで開かれていないか確認してください
- EPPlus は **非商用ライセンス** で動作します。商用利用の場合は EPPlus 商用ライセンスが必要です

### Q. MySQL に接続できない

- ポート番号のデフォルトは `3306` です
- `Allow User Variables=True` が必要な場合は接続文字列に追加してください

### Q. パイプラインが途中で止まる

ログパネルのエラーメッセージを確認してください。
各ステップはエラーが発生してもログに記録して続行します。
